## 1. 产品概述

`搬箱计划（Sokoban Project）` 是一款面向 PC、基于 Godot 4.6 开发的 2D 推箱子解谜游戏。

产品以经典 Sokoban 规则为基础，在保持低上手门槛的同时，加入了更具设计空间的扩展机制：

- 多色箱子
- 多色目标点
- 中性槽
- 面向玩家开放的内置关卡编辑器

它既是一款可直接游玩的成品游戏，也是一套可持续产出内容的轻量创作工具。

## 4. 核心卖点

## 4.1 多色箱子机制

本作最大的玩法差异点不是单纯复刻经典推箱子，而是在标准规则上扩展出多色匹配系统：

- 箱子可分为多种颜色
- 目标点可分为多种颜色
- 只有颜色匹配的箱子与目标点才能形成有效完成态
- 中性槽可以接收任意颜色箱子

这使关卡设计不再只围绕“位置”，而是同时围绕“顺序、分配、颜色关系”展开。

## 4.2 内置关卡编辑器

编辑器不是开发附属工具，而是产品正式功能的一部分。

玩家可以直接在游戏中：

- 新建关卡
- 绘制地形、箱子、目标点与玩家位置
- 配置多色内容
- 测试游玩
- 调用求解器验证可解性
- 保存为本地用户关卡
- 导入导出 JSON / XSB / 分享码

## 4.3 官方内容 + 玩家内容并存

本作既提供完整的官方关卡内容，也支持玩家自制内容接入主流程。

这保证了：

- 新玩家进入后就有足够内容可玩
- 高阶玩家在通关后仍有继续创作和分享的动力

## 5. 核心玩法说明

## 5.1 基础规则

- 玩家每次沿四方向移动一格
- 玩家不能穿墙
- 玩家可以推动箱子，但不能拉动箱子
- 当箱子前方为空时，玩家可将其向前推动一格
- 所有箱子都处于合法目标点完成态时，关卡完成

## 5.2 扩展规则

- 箱子具有颜色属性
- 目标点具有颜色属性
- 中性槽可接受任意颜色箱子
- 非匹配颜色即使占据目标点，也不算完成

## 5.3 辅助操作

- Undo
- Redo
- Restart
- Pause

产品默认鼓励玩家尝试、复盘和优化路线，而不是惩罚试错。

## 9. 技术实现特征

## 9.1 技术栈

- 引擎：Godot 4.6
- 语言：GDScript 为主
- 渲染：Mobile renderer
- 存档：本地 JSON
- 本地化资源：统一使用 JSON，便于社区翻译协作

## 9.2 核心工程特点

- 玩法数据层与渲染层分离
- 基于 Autoload 的全局服务管理
- 官方关卡与用户关卡统一索引
- 编辑器与游戏主流程互通
- 支持后台求解器验证
- 支持 headless 自检脚本

## 9.3 求解器能力

- IDA* 求解
- 死格剪枝
- 多色规则适配
- WorkerThreadPool 后台运行
- 可用于最优解验证与评分基线生成
