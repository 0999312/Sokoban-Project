# 关卡设计指南（Level Design Guide）

> 适用于 Sokoban v1.0 官方 50 关与玩家自制内容评审。

## 1. XSB 字符集（标准）

```
#  墙
   地板（空格）
.  目标点
$  箱子
*  箱子在目标点上
@  玩家
+  玩家在目标点上
-  外部空区（透明，可省略，等同 #）
```

注意：行末空格请保留；不要使用 Tab；外围必须由墙完整包围（可用 `-` 表示透明区域，导出 JSON 时会变为 `#`）。

## 2. 难度梯度建议

| 难度 | 箱子数 | 推荐尺寸 | 最优步数 | 设计要点 |
|---|---|---|---|---|
| 1 教学 | 1-2 | ≤ 8×6 | ≤ 15 | 单一概念引入，路径清晰 |
| 2 进阶 | 2-3 | ≤ 10×8 | ≤ 30 | 顺序依赖，避免角落死锁 |
| 3 精选 | 3-5 | ≤ 14×10 | ≤ 80 | 多解但有最优；强迫预规划 |
| 4 挑战 | 5+ | ≤ 18×14 | ≤ 200 | 长解、回溯多、空间敏感 |

## 3. 设计原则

1. **单一新概念**：每关只引入一个新挑战（例如"必须先推后侧箱子"）
2. **可读性**：通关路径总能在 1-2 屏内被完整观察
3. **公平性**：所有死锁状态对玩家在视觉上明确可识别
4. **优雅解**：最优解步数是 Solver 验证后的唯一权威值，不允许猜测
5. **避免噪声**：删除任何对解题无影响的多余地板/装饰

## 4. 设计流程（每关）

1. 在 `levels/templates/blank.xsb` 上手绘草图
2. 用本指南检查规则：1 个 `@`、`$` 数 = `.` 数、外围闭合
3. 把 `.xsb` 拷入 `levels/official/wN/NN.xsb`
4. 启动游戏 → 编辑器 → "导入 XSB" → "求解验证"
5. 把 Solver 给出的 `optimal_steps`/`optimal_pushes` 填入 JSON `metadata`
6. 玩家亲身通关一次，记录"主观难度"，与设计目标对比
7. 提交 PR：附 XSB、JSON、Solver 输出截图
